חדש: customer-office.service

src/app/service/customer-office.service.ts חדש ✨

שירות קל שמרכז את הלוגיקה של "האם הלקוח מאומת? אם לא — פתח דיאלוג".

@Injectable({ providedIn: 'root' })
export class CustomerOfficeService {

  /** מחזיר officePlace אם מאומת, אחרת פותח דיאלוג ומחכה */
  async getOrVerifyOffice(customer: User): Promise<Place | null> {
    if (customer.officePlace && customer.addressVerified) {
      return customer.officePlace
    }
    const result = await openDialog(CustomerDetailsValidateComponent,
      x => x.args = { customerId: customer.id, hint: 'הזן את הכתובת של המשרד שלכם' })
    return result?.officePlace ?? null
  }
}

עריכה: orderPaymentCalculator — חוק "+30 ₪"

src/app/order/orderPaymentCalculator.ts חוק חדש 📐

הוספת בדיקה מעל כל ה־switch הקיים. אם תנאי המשרד מתקיים — לוקחים מסלול חישוב מקוצר. אחרת — הקוד הקיים רץ כרגיל (אפס סיכון לרגרסיה).

הלוגיקה החדשה

// מאזורי "המשרד הקרוב" — מקבלים את חוק ה־+30
const nearAreas = [
  AreaType.גוש_דן_הרצליה_ראשון_לציון.id,
  AreaType.שפלה_קרוב.id,
  AreaType.שרון_קרוב.id
]

const isOfficePickup  = o.fromPlace?.isOurOffice
const isOfficeDelivery = tos.some(p => p.isOurOffice)
const officeArea = isOfficePickup ? o.fromPlace.area
                  : isOfficeDelivery ? tos.find(p => p.isOurOffice)!.area
                  : null

if (officeArea && nearAreas.includes(officeArea.id)) {
  // 🆕 חישוב מקוצר: צד אחר במלא + 30 (×multi)
  const otherSide = isOfficePickup ? tos[0].area : o.fromPlace.area
  result.items.push({
    desc: otherSide.caption,
    value: this.getPrice(otherSide) * o.priority.multiplier
  })
  result.items.push({
    desc: `המשרד שלנו (${officeArea.caption})`,
    value: 30 * o.priority.multiplier
  })
  result.total = result.items.reduce((s, i) => s + i.value, 0)
  return result
}

// אחרת — נמשיך ל־switch הרגיל הקיים, ללא שינוי

⚠️ נקודות סיכון / לאישור

📍
שדה Place.isOurOffice — האם נוסיף שדה boolean חדש ל־Place שמסמן "זה המשרד"? זה מאפשר לקלקולטור לזהות מי המשרד בלי לקרוא חזרה ל־customer.officePlace.id === place.id. נדרש אישור.
🔄
השתלת officePlace ב־Place הזמנה — האם להעתיק את כל השדות (clone) או לקשר ב־FK? clone יוצר אי־תלות מהמשרד (אם הלקוח שינה את המשרד אחר כך, הזמנות ישנות לא ישתנו). המלצה: clone.
🛡️
הזמנות קיימות — לקוחות בלי officePlace ימשיכו לעבוד כרגיל. ה־checkbox פשוט לא יעשה כלום עד שיש להם משרד. אין מיגרציה נדרשת.
🔁
"שמור והמשך" — בדיאלוג, אם הלקוח לחץ "ביטול" → לא נשמר כלום, ה־checkbox ב־order-details חוזר להיות לא מסומן.
🎯
שלב עתידי (לא בסקופ עכשיו): אימות נייד אמיתי דרך SMS — הקומפוננטה app-mobile-verify-field תהיה מוכנה כתשתית, רק נחבר ספק SMS כשתאשר.
