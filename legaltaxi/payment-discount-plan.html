🧩 ה־helper המוצע

private applyDiscountRow() {
  // 1) הסר שורת הנחה קודמת אם קיימת
  const lines = (this.payment.rows || '').split('\n')
                  .filter(l => !l.trim().startsWith('הנחה'))

  // 2) הוסף שורה חדשה לפי discountType
  if (this.payment.discountType === DiscountType.percent && this.payment.discount > 0) {
    const sumOriginal = lines.reduce((s, l) => s + parseFloat(l.split(':')[1] || '0'), 0)
    const discountValue = this.payment.discount
    const discountAmount = +(sumOriginal * discountValue / 100).toFixed(2)
    lines.push(`הנחה (${discountValue}%): -${discountAmount}`)
  }
  else if (this.payment.discountType === DiscountType.fixed && this.payment.discount > 0) {
    lines.push(`הנחה: -${this.payment.discount}`)
  }

  this.payment.rows = lines.join('\n')
}

⚠️ נקודות סיכון / שלא נשכח

📊
הנחה באחוז — על מה היא מחושבת? הצעתי לחשב על סכום ה־rows המקוריות (לפני הנחה). זה יוצא יותר נכון משימוש ב־originalAmount שיכול להכיל הנחות ישנות אם נערך כבר. אם תעדיף חישוב על originalAmount — תאמר.
🔄
תיקון מקרה קיים: הזמנות שכבר נשמרו עם discount > 0 אבל בלי discountType ובלי שורת הנחה ב־rows — אין מיגרציה אוטומטית. נשארות לא מסונכרנות עד עריכה ידנית. לאשר שזה בסדר.
🔍
זיהוי שורת הנחה נעשה בעזרת startsWith('הנחה'). לוודא שאין שורות אחרות בקלקולטור שמתחילות במילה "הנחה" (לא ראיתי כאלה ב־orderPaymentCalculator).
🏷️
שדה suffix ב־DiscountType: משמש את displayValue של Payment.discount כדי להציג '10%' או '50 ₪' או '0' לפי הסוג. בלי לקרוא לפונקציה נוספת.
