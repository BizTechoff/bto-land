01

עקרונות הליבה - חובה

🧱 מודולריות לפני הכל

כל יחידה (Module) - מוגדרת, מבודדת, ובעלת ממשק יחיד וברור. אפשר להחליף, להרחיב, או להוציא כל מודול - בלי לפגוע בשאר. כל פעולה עוברת דרך Module Interface, אף פעם לא דרך גישה ישירה לפרטים פנימיים.

🔗 השתלבות עתידית - לא רק POC

הקוד שייבנה ב-30 השעות מהווה היסוד של מלאך הזכויות המקיף. כל ישות, מודול, ושירות מתוכנן כך שיתאים לארכיטקטורה הסופית (DSL Engine, Digital Twin, Right Forge, Proactive Layer) - בלי כתיבה מחדש בעתיד.

📐 ארכיטקטורה תקנית של src/

app (common, route, service) · assets · server (index, server, jwt, remult-api, json-body) · shared (entity, enum, controller, type) - כפי שנקבע מראש, ללא חריגות.

03

מודולי ליבה (POC) + מסלול עתידי

@citizen
זהות האזרח. אוסף פרטי המגיש - עיר, תעודת זהות, תאריכי שירות, גודל דירה, פרטי בנק.
Digital Twin מלא - היסטוריה, זכויות פעילות, וכל המידע הקבוע.
@rights-registry
מאגר זכויות. בשלב זה - זכות אחת בלבד (הנחת ארנונה למילואים) מוטמעת hardcoded.
מנוע DSL גנרי - כל זכות מוגדרת כ-RightDefinition JSON.
@calc-engine
מנוע חישוב. הנוסחה מהאקסל - 3 שנים (2023-2025), 5% לחייל, מוגבל 100 מ"ר ראשונים, פרו-רטה ימי שירות.
Interpreter גנרי - מקבל RightDefinition + DigitalTwin ומחזיר תוצאה.
@authority
מאגר רשויות. טעינת קובץ ה-1,267 יישובים מ-Excel ל-PostgreSQL. lookup לפי שם עיר → אימייל גזבר.
Authority Adapters - לכל עירייה adapter ייעודי (PDF templates, API).
@document
יצירת מכתב. תבנית המכתב מהאקסל + הזרקת נתוני האזרח + יצירת HTML/PDF.
Template Engine - תבניות שונות לסוגי זכויות, רב-לשוני.
@signature
חתימה דיגיטלית. Canvas signature pad. שמירה כ-PNG ב-DB + צירוף ל-PDF.
חתימה תקפה משפטית עם audit log + timestamp + IP.
@payment
תשלום. אינטגרציה ל-Payme - 18 ₪ קבוע. webhook לעדכון סטטוס.
תמחור גמיש לכל זכות + תשלום מנוי + ערכות.
@mailer
שליחת אימייל. שירות SendGrid/SES - שולח את המכתב + ה-PDF לאימייל הגזבר.
Multi-channel: Email, WhatsApp, SMS, מערכת תיוק.
@audit
יומן ביקורת. כל פעולה (יצירה, חתימה, שליחה) נרשמת ב-audit_log. מהיום הראשון.
בסיס לכל ניהול תהליכי השגות, ערעורים, ומחלוקות.

05

השתלבות עתידית במלאך הזכויות המקיף

כל מודול ב-POC נכתב כך שיוכל להיכלל בפלטפורמה המלאה ללא כתיבה מחדש. הטבלה למטה ממפה את המעבר מ-POC אל הגרסה המקיפה:

@citizen מאחסן את 4 השאלות מהשאלון בלבד.
@citizen מכיל את ה-Digital Twin המלא - זהות, כל הזכאויות, היסטוריה, מסמכים.
@calc-engine מחשב hardcoded לארנונה+מילואים.
@calc-engine הופך ל-DSL Interpreter שמריץ כל RightDefinition.
@rights-registry מכיל זכות אחת.
@rights-registry מנוהל דרך Right Forge - AI שמכניס זכויות חדשות מקריאת חוקים.
@mailer שולח לאימייל גזבר.
@mailer הופך ל-Multi-channel: Email + WhatsApp + SMS + ערוצים נוספים.
@authority - lookup פשוט.
@authority הופך לאוסף של Authority Adapters - לכל עירייה adapter ייעודי.
@audit מתעד פעולות בסיסיות.
@audit הוא היסוד של ה-Proactive Layer + Appeals Flow.
אין מעקב פרואקטיבי.
Deadline Watcher רץ ברקע - מעקב אחרי תגובת גזברויות, השגות, ערעורים.
06

החלטות טכניות מרכזיות

📦 Database: PostgreSQL מהיום הראשון

כל ה-Entities נכתבות מהיום הראשון על Remult + PostgreSQL. לא SQLite. הסכמה מוגדרת multi-tenant ready (יש tenant_id בכל טבלה) - אפילו אם בשלב זה יש tenant אחד.

🔐 Auth: JWT + Email/Password מהיום הראשון

אזרח נכנס לשאלון - נוצר משתמש אנונימי עם session JWT. אחרי תשלום - האזרח יכול לשמור את האימייל שלו ולחזור. כל הטוקנים עם refresh token לפי הסטנדרט.

📝 Audit Log מהיום הראשון

כל פעולה (יצירת בקשה, חתימה, תשלום, שליחה) נרשמת ב-audit_log עם user_id, action, timestamp, ip, payload. זה לא "לעתיד" - זה מהיום הראשון, כי אי-אפשר להוסיף audit retroactively.

🌐 i18n מהיום הראשון

גם אם השלב הזה בעברית בלבד - כל מחרוזת UI עוברת דרך i18n service. ברגע שנצטרך אנגלית/ערבית - אין צורך לכתוב מחדש שום מסך.

📤 Deploy: דומיין + SSL מהיום הראשון

גם ה-POC עולה לאוויר עם דומיין רשמי (rights-angel.co.il או דומיין שיבחר) + HTTPS. הלקוח רואה את ה-POC ככלי פרודקשן מהרגע הראשון.
