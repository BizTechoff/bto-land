1החזון

מערכת הפעלה אחת לכל חיי בית חב״ד — כל הקהילה במקום אחד.

שליח חב״ד מנהל קהילה שלמה בעצמו: אנשים, אירועים, תרומות, לוח עברי, מבצעים, דיוור ומסמכים — לרוב בעשרה כלים שונים שלא מדברים זה עם זה. שלוח מאחד את הכול לפלטפורמה אחת מודולרית, בעברית כברירת מחדל ובאנגלית, שמכירה את הלוח העברי ואת זמני היום לפי מיקום בית חב״ד.

3מבנה התיקיות בקוד

תבנית דומיין עקבית — אותה צורה חוזרת בכל מודול, כך שכל חדש מתמצא מיד.

src/
├─ app/
│  ├─ common/                         # רכיבים גנריים משותפים
│  ├─ route/                          # app-routing
│  ├─ service/                        # שירותי רוחב
│  └─ [side-bar-item]/             # דומיין = פריט בתפריט הצד
│     ├─ [item]-list.component
│     ├─ [item]-details.component
│     ├─ [item]-selection.component
│     ├─ [item]-selection-field.component
│     └─ [item].service.ts            # component → service → controller
├─ assets/
│  └─ i18n/  he.json · en.json        # עברית ברירת מחדל + אנגלית
├─ server/
│  ├─ index.ts · server.ts
│  ├─ jwt.ts · remult-api.ts · json-body.ts
│  └─ s3.ts · geo.ts · email.ts · heb-cal.ts · ai.ts   # ממשקים
└─ shared/
   └─ [side-bar-item]/                # דומיין — אותה חלוקה כמו app/
      ├─ [item].entity.ts             # extends IdEntity
      ├─ [item].enum.ts               # @ValueListFieldType
      ├─ [item].request.type.ts       # interface · בקשה
      ├─ [item].response.type.ts      # interface · תשובה
      └─ [item].controller.ts         # ControllerBase · @BackendMethod
קונבנציית FK: שדה מחרוזת eventTypeId לצד יחס @Relations.toOne(() => EventType) בשם eventType — מזהה לשמירה, יחס לתצוגה.

9שכבת AI — בינה לאורך הפלטפורמה

לא מודול בתפריט — יכולת אופקית שכל דומיין צורך, דרך שער ספק אגנוסטי. עם עיקרון‑על מקודש: אדם בלולאה.

🛡️ HUMAN IN THE FLOW — עיקרון‑על

כל פעולה שה‑AI מציע דורשת אישור ידני מהשליח — תמיד. ה‑AI מנסח, מציע ומסכם — אך לעולם לא שולח, מוחק או מבצע ללא לחיצת אישור. החריג היחיד: אם השליח סימן והגדיר מראש אוטונומיה לנושא ספציפי — ורק בו — ה‑AI ימשיך לבד, עם תיעוד מלא.
